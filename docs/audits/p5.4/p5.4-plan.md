## Phase specs (b→e)

### P5.4b — Home dispatcher redesign
Goal: `/` reads as a beautiful launchpad, not a scattered board. Reorganize the cards that lead to
other surfaces; centralize; scale type; use the reclaimed real estate (no more LeftRail); Masthead
head; keep the live pulse KPIs (with their P5.3d verdict words) but make them sing. Research
dispatcher/bento/launchpad layouts (ultracode). Remove the PWA/apple install prompt if one exists.
Stay strictly within yesid.dev branding (tokens, grid, glow, motion). One h1 + dot.

### P5.4c — DetailShell + metrics/status port (super-important per operator)
Read `../yesid.dev/apps/web/src/lib/components/{projects,blog}/*DetailPage.svelte` EXACTLY. Build one
faithful **DetailShell** (promotion-ready): full-bleed detail header band (dot-grid `.detail-header-grid`
+ upright mono edge-metadata + back-link → display h1 + dot → lede → pills → QuietMode) + edge-to-edge
`<Separator variant="hazard">` + 3-col grid (`1fr 2fr 1fr` @1024, gap 2rem; 2-col when no right rail) +
sticky rails `top: var(--chrome-offset)` + `observeActiveToc` feeding TocNav + TocPill + optional
CtaBand. Re-seat **metrics** (header=detail header, left=metrics ToC, center=per-metric
CollapsibleSections, right=Provenance/Coverage/Freshness) and **status** (left=lanes ToC, center=lanes,
right=per-feed stat cards) onto it. **Delete** MetricsExplainer's inlined grid + hardcoding, and delete
`layout/DetailTemplate.svelte`. Faithful port, no lookalike hardcoding.

### P5.4d — SurfaceRail + ScheduleTable
- **SurfaceRail** (build-local, promotion-ready): sticky left rail = GrainPicker (reuse) + FilterGroup
  (port from yesid `shared/FilterGroup` + FilterSummary + ListingMobileFilters — transit lacks these;
  replaces the 5+ per-feature filter-pill reimplementations) + TocNav (reuse). `top: var(--chrome-
  offset)`, FOCUS/quiet-mode aware (S10), collapses to a mobile drawer + TocPill. Apply to
  line-reliability, stop-reliability, network, hotspots, alerts — replacing the floating top grain bar.
- **ScheduleTable** (build-local): one component, modes `grid` (column-major time grid — the StopDetail
  idiom) + `board` (route · time · delay-glyph — the next-departures idiom), `cap`+more, AbsentValue
  empty state, shared status-scale delay coloring. Apply to stop schedule + line schedule (RouteDetail
  schedule tab) + next-departures board. Schedules render as a table everywhere.
